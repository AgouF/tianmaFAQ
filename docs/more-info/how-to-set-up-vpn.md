# VPN怎么搭建

## VPN搭建的基本原理

VPN（虚拟专用网络）的核心原理是在公共互联网上创建一个加密的“隧道”，将您的设备与远程服务器安全连接起来。当您连接到VPN服务器后，您的网络流量会通过这条加密通道传输，对外显示的是VPN服务器的IP地址，从而保护您的真实身份和位置，并可能访问受地域限制的内容。

搭建VPN主要涉及两个关键组件：**VPN服务器**和**VPN客户端**。服务器部署在您选择的网络位置（如云端VPS、家庭路由器或数据中心），负责接收、解密您的请求，然后代表您访问互联网，并将加密的响应传回。客户端则是安装在您设备上的软件，负责发起连接并处理加解密。

## 选择适合的VPN协议与技术

选择合适的协议是搭建稳定、安全VPN的第一步。目前主流的选择包括：

**1. WireGuard**：这是目前最受推荐的新协议。它代码量小（约4000行），速度快，加密现代，连接稳定。对于个人和小型团队搭建，它是首选。
**2. OpenVPN**：老牌、成熟、高度可配置的协议，经过长时间的安全考验。配置稍复杂，但兼容性极佳。
**3. IPSec/IKEv2**：通常内置于操作系统（如iOS、macOS），连接速度快，尤其在移动设备切换网络时非常稳定。

对于希望获得更佳网络体验的用户，也可以探索一些集成了代理和加速技术的方案。例如，您可以参考这份详细的 **[如何搭建魔法](/network/vpn/how-to-build-magic)** 指南，其中介绍了一种集成了先进传输协议和伪装技术的流行工具，能有效提升在复杂网络环境下的连接成功率与速度。

## 逐步搭建VPN服务器（以WireGuard为例）

以下是使用性价比高的云服务器（如Vultr、DigitalOcean、阿里云国际版等）搭建WireGuard VPN的核心步骤：

**### 第一步：准备服务器**
1.  购买一台海外Linux VPS（推荐Ubuntu 22.04系统）。
2.  通过SSH连接到您的服务器。

**### 第二步：一键安装与配置**
目前有许多优秀的自动化脚本可以简化安装。例如，使用 `wg-easy` 或 `Angristan` 的脚本。
```bash
# 示例：使用 wg-easy 的 Docker 安装方式（需先安装Docker）
docker run -d \
  --name=wg-easy \
  -e WG_HOST=你的服务器公网IP \
  -e PASSWORD=你的管理密码 \
  -v ~/.wg-easy:/etc/wireguard \
  -p 51820:51820/udp \
  -p 51821:51821/tcp \
  --cap-add=NET_ADMIN \
  --cap-add=SYS_MODULE \
  --sysctl="net.ipv4.conf.all.src_valid_mark=1" \
  --sysctl="net.ipv4.ip_forward=1" \
  --restart unless-stopped \
  weejewel/wg-easy
```
安装后，访问 `https://你的服务器IP:51821` 即可通过Web界面轻松管理客户端和密钥。

**### 第三步：配置客户端**
在Web管理界面生成一个客户端配置（会生成一个二维码和`.conf`文件）。在您的手机或电脑上下载WireGuard客户端，扫描二维码或导入配置文件，然后开启连接即可。

## 重要安全与优化提示

1.  **防火墙设置**：务必在服务器防火墙和云服务商安全组中，只开放VPN服务所需的UDP端口（如WireGuard的51820）和管理端口（如51821），关闭不必要的端口。
2.  **定期更新**：保持服务器系统和VPN软件为最新版本，以修复安全漏洞。
3.  **备份配置**：定期备份服务器的WireGuard配置文件。
4.  **性能优化**：对于TCP流量，可以尝试在服务器上启用BBR拥塞控制算法来提升速度：
    ```bash
    echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf
    echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf
    sysctl -p
    ```

## 常见问题

### 搭建VPN合法吗？
在绝大多数国家和地区，出于保护个人隐私、安全访问公司内网等目的，搭建和使用VPN是**完全合法**的。但是，请务必遵守所在国和使用地的法律法规，**不得将VPN用于任何非法活动**，如访问明确禁止的违法内容、进行网络攻击等。

### 为什么我的VPN连接速度很慢？
速度受多种因素影响：
- **服务器位置**：选择物理距离相对较近、网络质量好的服务器。
- **服务器带宽**：确保你购买的VPS带宽充足。
- **本地网络**：你自身的宽带或移动网络质量是基础。
- **协议选择**：WireGuard通常比OpenVPN速度更快、延迟更低。
- **运营商干扰**：某些网络环境下，标准VPN端口可能被限速或干扰，这时可考虑使用 **[如何搭建魔法](/network/vpn/how-to-build-magic)** 中提到的、具备流量伪装能力的工具来改善。

### 一个VPN服务器可以多少人同时使用？
这取决于服务器（VPS）的CPU、内存和**带宽**。一个最低配置（1核1G）的VPS，如果带宽充足（如1Gbps），轻松支持10-20人日常浏览使用。如果用于高清视频流或大文件下载，同时在线人数就需要减少。关键在于监控服务器的带宽使用率和CPU负载，根据实际情况调整。
